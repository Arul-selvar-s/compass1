# Compass — Private Diary App

A diary app disguised as a working compass. Built with Kotlin, Jetpack Compose, Hilt, Room, and Google Drive sync. No local Android Studio needed — every build runs on GitHub Actions.

## How to upload this project

1. Create a **new, empty** GitHub repository (or delete all files from your existing one).
2. Upload **every file and folder** in this ZIP to the root of that repository — including the hidden `.github` folder.
   - If drag-and-drop on the GitHub web UI doesn't show the `.github` folder, use **Add file → Create new file** and type the path `.github/workflows/build.yml` directly — GitHub will create the folders for you, then paste the file content.
3. Once uploaded, go to the **Actions** tab. A build will start automatically (or click **Run workflow** if it doesn't).
4. Wait for the green checkmark (3–4 minutes), then open the finished run → **Artifacts** → download `Compass-App-APK`.
5. Unzip it, install `app-debug.apk` on your phone (you may need to allow "install unknown apps" once).

## First-time setup in the app

1. Open the app → **Get Started**
2. Type any number 0–360 as your **secret angle** (this is your password)
3. *(Optional)* **Sign in with Google** to enable Drive backup
4. Done — your diary is ready

## How to unlock every time

1. Open the app — you'll see a real compass
2. **Tap the angle number** at the bottom
3. Type your secret angle **exactly**
4. Tap **Find Direction** — the compass spins, points to your angle, and opens your diary if correct

## Setting up Google Drive sync (one-time, optional)

The app can back up your diary to your own Google Drive so you can restore it on a new phone. This requires a one-time setup in Google Cloud Console — **no Firebase or `google-services.json` needed**, since the app only uses basic Android sign-in + the Drive REST API.

1. Go to **[console.cloud.google.com](https://console.cloud.google.com)** → **New Project** → name it `Compass`
2. **APIs & Services → Library** → search **Google Drive API** → **Enable**
3. **APIs & Services → OAuth consent screen** → choose **External** → fill in app name + your email → **Save and Continue** through each step
4. **APIs & Services → Credentials → Create Credentials → OAuth client ID**
   - Application type: **Android**
   - Package name: `com.compass.diary`
   - SHA-1 fingerprint: get this from your GitHub Actions build log (the debug keystore is auto-generated by Gradle on first build — you can print it by temporarily adding a step `run: keytool -keystore ~/.android/debug.keystore -list -v -storepass android` to the workflow and reading the log)
5. Click **Create**. That's it — no file needs to be added back to the repo.
6. Rebuild and reinstall the app. Google Sign-In will now actually authorize Drive access.

Until this is done, "Sign in with Google" will still let you sign in (to save your email), but the Drive upload/restore calls will fail silently and you'll see "Sync failed" in Settings — this is expected and not a bug.

## What's fixed in this build

- **Formatting (Bold/Italic/Underline/Strikethrough)** — now persists correctly. Formatting is tracked as a separate list of spans and re-applied to the text on every render, since the keyboard strips styling on every keystroke by design.
- **Saving on old/new dates** — fixed a bug where saving silently failed if no database row existed yet for that date.
- **Exact password match** — the secret angle must now match exactly (within 0.5° for float rounding only), not an approximate range.
- **Save & Lock** — tap the 💾 icon to permanently lock your current entry. Locked text becomes read-only; you can keep adding new text below it, and saving also triggers a Google Drive backup.
- **Google Drive** — sign-in now requests the `drive.file` scope and the app uploads/downloads a real backup file, not just a cosmetic sign-in screen.
